# NotifReader — приложение-перехватчик уведомлений

Это готовый проект Android Studio. Здесь я НЕ смогла собрать сам .apk (в этой
среде нет Android SDK и нет доступа в интернет для его скачивания), но проект
полностью рабочий — собрать APK можно за 2–5 минут на своём компьютере.

## Что делает приложение
- Экран с кнопкой "Дать доступ к уведомлениям" — открывает системные настройки,
  где нужно один раз включить доступ для NotifReader.
- После этого сервис `NotifListenerService` перехватывает АБСОЛЮТНО ВСЕ пуш-
  уведомления со всех приложений телефона (это официальный Android API
  `NotificationListenerService`, других легальных способов читать чужие
  уведомления не существует).
- Все перехваченные уведомления (имя приложения, заголовок, текст, время)
  показываются списком прямо в приложении, самые новые — сверху.

## Как собрать APK

### Вариант 0 — GitHub Actions (без установки чего-либо на компьютер) ⭐
В проекте уже лежит готовый workflow `.github/workflows/build.yml`, который
сам соберёт APK в облаке при каждой загрузке кода.

1. Зайдите на https://github.com и создайте новый **пустой** репозиторий
   (например, `NotifReader`), ничего в него сразу не добавляя (без README).
2. Загрузите в него содержимое этой папки одним из способов:
   - через веб-интерфейс GitHub: "uploading an existing file" → перетащите
     все файлы и папки (включая скрытую папку `.github`);
   - или через git в терминале:
     ```bash
     cd NotifReader
     git init
     git add .
     git commit -m "initial commit"
     git branch -M main
     git remote add origin https://github.com/ВАШ_ЛОГИН/NotifReader.git
     git push -u origin main
     ```
3. Откройте на GitHub вкладку **Actions** в вашем репозитории — сборка
   запустится автоматически (займёт 2–4 минуты).
4. Когда сборка завершится (зелёная галочка), откройте этот запуск и внизу
   страницы в разделе **Artifacts** скачайте `NotifReader-debug-apk.zip`.
5. Распакуйте — внутри будет `app-debug.apk`. Перекиньте на телефон и
   установите.

Если сборка запустилась не сама — зайдите в Actions → выберите workflow
"Build APK" → кнопка "Run workflow" (это возможно благодаря строке
`workflow_dispatch` в файле workflow).

### Вариант 1 — Android Studio
1. Скачайте и установите Android Studio: https://developer.android.com/studio
2. File → Open → выберите папку `NotifReader` (эту, где лежит `build.gradle`).
3. Studio сама предложит скачать нужный Gradle Wrapper и Android SDK — 
   согласитесь (нужен интернет).
4. Дождитесь окончания синхронизации (Gradle Sync).
5. Build → Build Bundle(s) / APK(s) → Build APK(s).
6. Готовый файл появится в
   `app/build/outputs/apk/debug/app-debug.apk`
7. Скопируйте этот файл на телефон и установите (разрешите установку из
   неизвестных источников).

### Вариант 2 — из терминала (если уже есть Android SDK)
```bash
cd NotifReader
./gradlew assembleDebug
```
APK окажется там же: `app/build/outputs/apk/debug/app-debug.apk`

## Важно про разрешения
Доступ к чтению чужих уведомлений (`NotificationListenerService`) — это
защищённая системная функция. Пользователь ОБЯЗАН вручную включить её в
Настройки → Приложения → Специальный доступ → Доступ к уведомлениям.
Это нельзя получить автоматически или в фоне — так задумано Google
специально для защиты приватности.

## Структура проекта
- `MainActivity.kt` — экран со списком и кнопкой запроса доступа.
- `NotifListenerService.kt` — сервис, который ловит уведомления.
- `NotifStore.kt` — хранилище уведомлений в памяти (сбрасывается при
  перезапуске приложения; при желании легко заменить на базу Room, чтобы
  хранить историю постоянно).
- `NotifAdapter.kt` — адаптер списка (RecyclerView).
